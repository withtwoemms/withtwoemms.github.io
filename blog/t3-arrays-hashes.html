List ⊃ Array ⊃ Hash

April 2nd 2015

Let's keep things simple: a hash is a type of array and an array is a type of list. A list is a sequence of values. The emphasis there is important to the foundational notion of a list. That is, a list is a collection of values in some specified order. Watch this:

  1. 1, 2, 3, 4
Fig. 1: A list is a sequence of values
...and *BAM*: list. Simple as that. I only added the commas for readability. The sequence listed is still a list sans commas. The commas are reading aids called "delimiters" that help to separate the different values (a space is also one of these). This is a very convenient example where the collected elements just happen to be in order. Note that this collection would still be a list if the elements were not in numerical order. For instance:
  1. 3, 1, 4, 2
Fig. 2: Numerical order doesn't matter
This is so because the sequence actually refers to the notion that each element has something before and after it. Both examples fulfill the definition of a sequence regardless of ordering by value. Plain ol' list implementations have a "push" and a "pop" method defined in the very least. These methods, allow for access to list elements by adding to or removing from the end of the list, respectively. Such a list is colloqially called a stack as the last item in is the first item out---like a stack of paper, but I digress..

Let's talk about something more applicable...

An array is a list whereby elements of the list can be accessed by an index. That is, elements have an identifying digit that, when referenced, can by used to return the element itself. This type of element access is called "random access" since element-getting is not only sequential. In Ruby, as in many other languages, that index begins with 0. This is so-called "0-based numbering." The access operator for retrieving an array element is typically a method of the form: [index]. Calling such a method gives the following:

  1. array = Array.[](3,1,4,2) [3,1,4,2]
  2. array.[](0) 3
  3. array.[](2) 4
Fig. 3: Array instantiation and access
In Fig. 3, an array is first created and then the 0th and 2nd elements are accessed.
  1. array = [3,1,4,2] [3,1,4,2]
  2. array[0] 3
  3. array.[2] 4
Fig. 4: Alternate (simpler) way of creating and accessing arrays

From the rather boring list, to the more agile array, it can be seen that much convenience is gained by allowing random access to a elements within a collection. The wanted value can be retrieved without having to manipulate the collection. Great. Cool. There's more. The integer indicies used to access values in an array don't convey a lot of meaning. What if values in the array represented somthing like the 'number of items in a grocery cart'? Instead of the indicies that are associated with each value, other objects--strings in this instance--can be used to refer to each value. Such an array is known as an associative array because some piece of identifying information is associated with each value.

  1. hash = Hash.new {}
  2. hash.store("apples",3)
  3. hash.store("onion",1)
  4. hash.store("bananas",4)
  5. hash.store("oranges",2)
  6. hash["apples"].assoc ["apples",3]
  7. hash["onion"]1
  8. hash.keys ["apples","bananas","onion","oranges"]
  9. hash.values [3,1,4,2]
Fig. 5: A "hash" is an associatve array. *NOTE* the simpler notation
The piece of identifying information that allows access to a value is called a key. In this respect, the index of an array element is a key and so too are the food item names used in Fig. 5. Deep down in the definition of an accociative array, there is a special function that connects a key to its value. This function is known as a hash function, hence the monicker "hash" for associative array.

In ruby, implementation of a list data structure is the array. These are good for storing values to which random (or sequential) access is needed. Such as when the units of data are reflected within the program--e.g. collecting sensor output every 5 milliseconds. In such a scenario, each extracted value can be related to every other. The hash is useful when this is not the case such as in marking an event. This allows that event to referenced uniquely. For example, if the sensor is only triggered to record when something happens. The hash can then be used to store the time of the event and the value.
